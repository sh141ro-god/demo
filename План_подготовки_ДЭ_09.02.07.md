## ЧТО КОНКРЕТНО НУЖНО СДЕЛАТЬ — ПО ПУНКТАМ

### Модуль 1 — База данных средствами СУБД (≈50 мин)
- [ ] Создать БД в выбранной СУБД на основе предметной области (Приложение 1).
- [ ] Соблюсти **3-ю нормальную форму** с обеспечением **ссылочной целостности**.
- [ ] Согласованная схема именования, корректные **первичные и внешние ключи**.
- [ ] Построить **ER-диаграмму** (таблицы, связи, атрибуты, ключи) и выгрузить в **PDF**.
- [ ] Подготовить и **импортировать данные** из предоставленных файлов (пометка import, Приложение 2).
- [ ] Сохранить результат: **скрипт БД** (или .dt для 1С).

### Модуль 2 — Алгоритм и создание приложения (≈40 мин)
- [ ] Составить **алгоритм в виде блок-схемы** по **ГОСТ 19.701-90**, выгрузить в **PDF**.
- [ ] Единый стиль интерфейса по руководству (Приложение 3): корректные **заголовки окон**, **иконка приложения**, **логотип компании** на главной форме (из ресурсов).
- [ ] Стиль кода: осмысленные идентификаторы, **CamelCase** (C#), не более одной команды в строке.
- [ ] Подключить БД к приложению; список товаров берёт данные из БД.
- [ ] **Окно входа — первое.** Логин/пароль из БД, либо вход **гостем** на просмотр товаров.
- [ ] Доступ к остальным модулям — только после успешной авторизации.
- [ ] Интерфейсы для **всех ролей**: гость, авторизованный клиент, менеджер, администратор.
- [ ] **Выход** обратно на окно входа из любой роли.
- [ ] **ФИО пользователя** в правом верхнем углу после входа.
- [ ] Карточка товара выводит: **фото** (если нет — заглушка `picture.png`), наименование, категория, описание, производитель, поставщик, цена, единица измерения, количество на складе, действующая скидка.
- [ ] **Подсветка строк/карточек:**
  - скидка **> 15%** → фон **#2E8B57**;
  - цена снижена → основная цена **перечёркнута, красный шрифт**, рядом итоговая цена **чёрным**;
  - нет на складе → строка/карточка **голубым**.
- [ ] Отладка; создать **docx со скриншотами** корректной работы.

### Модуль 3 — Последовательный UI, формы, удаление (≈1 ч 30 мин)
- [ ] Навигация между окнами/страницами, в т.ч. **кнопка «Назад»**; на каждом окне свой заголовок.
- [ ] **Обработка исключений**: окна сообщений типов ошибка/предупреждение/информация — с нужным заголовком, **пиктограммой** и информативным текстом (что не так и как исправить).
- [ ] Комментарии в коде — только для неочевидных мест.
- [ ] **Поиск** (менеджер, админ): по всем текстовым полям, по нескольким атрибутам одновременно.
- [ ] **Сортировка** по количеству на складе (возрастание/убывание).
- [ ] **Фильтр по поставщику**; первый пункт списка «**Все поставщики**» сбрасывает фильтр.
- [ ] Поиск/сортировка/фильтр — **в реальном времени, без кнопки**; фильтр и поиск **совместно**; выбранная сортировка **сохраняется** при фильтрации и поиске.
- [ ] **Добавление/редактирование товара** — только админ, в **отдельном окне-форме**.
- [ ] Переходы: **клик по товару** → редактирование; кнопка «**Добавить товар**» → добавление.
- [ ] Поля формы: фото, наименование, категория (**выпадающий**), описание, производитель (**выпадающий**), поставщик, цена, единица измерения, количество, скидка.
- [ ] **Валидация:** цена — сотые допустимы, **не отрицательная**; количество — **не отрицательное**.
- [ ] При редактировании поля **подгружаются из БД**.
- [ ] **ID:** при добавлении не показывается (авто **+1**), при редактировании — **только для чтения**.
- [ ] **Изображение:** добавить/заменить, сохранять в папку приложения, ограничение **300×200**, в БД хранить **путь**, при замене **удалять старое** фото.
- [ ] Запретить открытие **более одного окна редактирования**.
- [ ] **Удаление товара** (админ); **нельзя удалить товар, который есть в заказе**.
- [ ] После любого CRUD список товаров **обновляется**.

### Модуль 4 — Работа с заказами (≈1 ч, только профильный)
- [ ] Кнопка «**Заказы**» в интерфейсах менеджера и админа.
- [ ] Вывод заказов по макету: **артикул заказа, статус заказа, адрес пункта выдачи, дата заказа, дата доставки**.
- [ ] **Добавление/редактирование заказа** — только админ, в отдельном окне-форме.
- [ ] Переходы: клик по заказу → редактирование; «**Добавить заказ**» → добавление.
- [ ] Поля формы: артикул, статус заказа (**выпадающий**), адрес пункта выдачи, дата заказа, дата выдачи.
- [ ] **Удаление заказа** (админ); после CRUD список **обновляется**.
- [ ] Загрузить всё в **репозиторий системы контроля версий**: исходный код (структура с файлами, **не архив**), исполняемые файлы, скрипт БД, прочие графич./текст. файлы.

## Критичные детали для баллов (легко потерять)

- Точные цвета: фон скидки **#2E8B57**, отсутствие на складе — **голубой**, сниженная цена — **красный** перечёркнутый + итог **чёрным**.
- Гость и авторизованный клиент видят товары **без** поиска/сортировки/фильтра; они только у менеджера и админа.
- Редактирование/добавление/удаление товаров и заказов — **только администратор**.
- ID: авто +1 при добавлении, read-only при редактировании.
- Картинка-заглушка `picture.png`, иконка приложения, логотип — заранее положить в ресурсы.
- Два PDF (ER-диаграмма, блок-схема ГОСТ) и docx со скриншотами — это отдельные обязательные артефакты, не забыть выгрузить.
