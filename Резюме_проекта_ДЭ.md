# Резюме проекта: приложение для демоэкзамена 09.02.07

Документ-передача для продолжения работы (в т.ч. в Cowork). Содержит контекст, архитектуру, соглашения по коду, разобранные ошибки, текущий прогресс и список оставшихся задач.

---

## 1. Контекст и цель

- **Цель:** подготовиться к демоэкзамену (ДЭ) по специальности **09.02.07 «Информационные системы и программирование»**, квалификация «Программист», и собрать проект.
- **Предметная область:** магазин обуви — учёт товаров на складах + заказы.
- **Подход:** репетиция полного проекта (Модули 1–4), чтобы быть готовым к любому уровню.

## 2. Стек

- **C# + WPF** (десктоп)
- **Entity Framework Core** (database-first, scaffold через EF Core Power Tools)
- **MS SQL Server** (инстансы на машине: `.` / `MSSQLSERVER` и `.\MSSQLSERVER01`)
- **Syncfusion** (community-лицензия) — опционально, для грида/полей/тем
- **CommunityToolkit.Mvvm** — для `ObservableObject`/`[ObservableProperty]` (по желанию)

## 3. Структура экзамена и требования

| Уровень | Модули | Время | Балл |
|---|---|---|---|
| ПА | 1–2 | 1ч30 | 25 |
| ГИА базовый | 1–3 | 3ч | 50 |
| ГИА профильный | 1–4 | 4ч | 75 |

- **Модуль 1 — БД:** 3НФ, первичные/внешние ключи, ER-диаграмма (PDF), импорт данных из Excel, скрипт БД.
- **Модуль 2 — приложение:** окно входа (логин/пароль из БД + гость), роли, список товаров, единый стиль, иконка/логотип, блок-схема (ГОСТ 19.701-90, PDF), docx со скриншотами.
- **Модуль 3 — UI/CRUD:** навигация + «Назад», обработка исключений (MessageBox), поиск/сортировка/фильтр, форма добавления/редактирования товара, удаление.
- **Модуль 4 — заказы (только профильный):** CRUD заказов, выгрузка в Git.

### Критичные детали (проверяют дословно)
- **Подсветка строк/карточек:** скидка > 15% → фон **#2E8B57**; нет на складе → **голубой**; цена снижена → старая цена **перечёркнута красным** + итог **чёрным**.
- **Роли:** гость и авторизованный клиент — только просмотр (без поиска/сортировки/фильтра); менеджер — просмотр с поиском/сортировкой/фильтром + просмотр заказов; администратор — полный CRUD товаров и заказов.
- **ФИО** в правом верхнем углу, кнопка **«Назад»**, гостевой режим.
- **Поиск** по всем текстовым полям (включая названия из навигаций), по нескольким атрибутам сразу; **фильтр по поставщику** с пунктом «Все поставщики»; **сортировка** по количеству на складе; всё **в реальном времени и совместно**, сортировка сохраняется при фильтре/поиске.
- **Форма** — в **отдельном окне**; ID при добавлении скрыт (авто +1), при правке read-only; валидация (цена/количество не отрицательные); категория/производитель — выпадающие списки; изображение 300×200, путь в БД, старое фото удалять; **запрет второго окна редактирования**; после CRUD список обновляется; **нельзя удалить товар, который в заказе**.
- **Артефакты:** ER-диаграмма (PDF), блок-схема (PDF, ГОСТ 19.701-90), docx со скриншотами, заливка кода в Git (структура с файлами, не архив).

## 4. Архитектура приложения

- **MainWindow** — хост. Содержит `ContentControl x:Name="MainContent"`. Навигация — **подмена `UserControl`** в этом ContentControl, со стеком истории для «Назад».
- **Экраны — это `UserControl`** (НЕ `Page`! Page нельзя класть в ContentControl).
- **Формы добавления/редактирования — отдельные модальные `Window`** (`ShowDialog()`): даёт блокировку (нет второго окна редактирования) + `DialogResult` для обновления списка.
- **Session** (статический класс) — общее состояние:

```csharp
public static class Session
{
    public static testContext Context { get; set; } = new testContext(); // общий контекст
    public static MainWindow Window { get; set; }                        // ссылка на хост
    public static User CurrentUser { get; set; }                         // null = гость

    public static bool   IsGuest  => CurrentUser == null;
    public static string RoleName => CurrentUser?.IdRoleNavigation?.Name ?? "Гость";
    public static bool   IsAdmin   => RoleName == "Администратор";
    public static bool   IsManager => RoleName == "Менеджер" || IsAdmin;
    public static void   Logout()  => CurrentUser = null;
}
```

- **MainWindow** задаёт `Session.Window = this;` в конструкторе и реализует навигацию:

```csharp
private readonly Stack<UserControl> _history = new();
public void Navigate(UserControl view)
{
    if (MainContent.Content is UserControl current) _history.Push(current);
    MainContent.Content = view;   // именно .Content, не сам ContentControl
}
public void GoBack() { if (_history.Count > 0) MainContent.Content = _history.Pop(); }
```

- Переход с экрана: `Session.Window.Navigate(new tableControl());`

- **Таблицы (3НФ):** Role, User, Category, Manufacturer, Supplier, Unit, OrderStatus, Product, Order, OrderItem (строки заказа — связь заказ↔товар, нужна для проверки удаления).
- **Скрипт схемы + тестовых данных:** файл `shoe_store.sql` (тестовые товары специально покрывают правила подсветки; товар Id=1 уже в заказе — удаление должно блокироваться).
- **Тестовые логины:** `admin/admin`, `manager/manager`, `client/client`.
- **Scaffold** через EF Core Power Tools → контекст `testContext`, модели с навигациями `IdXxxNavigation` и ключами `IdXxx`.
- **Нюанс DbContext:** Power Tools генерирует только конструктор с `DbContextOptions` (под DI). Для `new testContext()` нужно добавить в **отдельный partial-файл** параметрless-конструктор + `OnConfiguring`:

```csharp
public partial class testContext
{
    public testContext() { }
    protected override void OnConfiguring(DbContextOptionsBuilder b)
    {
        if (!b.IsConfigured)
            b.UseSqlServer("Server=.;Database=shoe_store;Trusted_Connection=True;TrustServerCertificate=True");
    }
}
```

## 6. Соглашения и паттерны кода

- **Загрузка списка:** `Include` по каждой навигации, затем `Local.ToObservableCollection()`:
```csharp
_context.Products
    .Include(p => p.IdCategoryNavigation)
    .Include(p => p.IdManufacturerNavigation)
    .Include(p => p.IdSupplierNavigation)
    .Include(p => p.IdUnitNavigation)
    .Load();
Products = _context.Products.Local.ToObservableCollection();   // ObservableCollection!
MainList.ItemsSource = Products;
DataContext = this;   // ОБЯЗАТЕЛЬНО для combobox-привязок через RelativeSource
```
- **Вложенная навигация:** `.Include(o => o.OrderItems).ThenInclude(oi => oi.IdProductNavigation)`.
- **Показ связанного значения:** биндить `Navigation.Name`, а не числовой id. Требует `Include`.
- **ComboBox по внешнему ключу внутри `DataTemplate`:**
```xml
<ComboBox DisplayMemberPath="Name"
          ItemsSource="{Binding DataContext.Categories, RelativeSource={RelativeSource AncestorType=ListBox}}"
          SelectedValue="{Binding IdCategory}" SelectedValuePath="Id"/>
```
  `Categories` — публичное **свойство** + `DataContext = this` (иначе пусто).
- **Поиск/фильтр/сортировка — через `CollectionView`, коллекцию НЕ мутировать.** Поиск и фильтр объединять в одном предикате:
```csharp
private string _searchText = "";
private int? _filterSupplierId = null;   // null = «Все»
private void ApplyFilter()
{
    var view = CollectionViewSource.GetDefaultView(MainList.ItemsSource);
    view.Filter = obj =>
    {
        if (obj is not Product p) return false;
        bool s = string.IsNullOrEmpty(_searchText)
            || (p.Name?.Contains(_searchText, StringComparison.OrdinalIgnoreCase) ?? false)
            || (p.Description?.Contains(_searchText, StringComparison.OrdinalIgnoreCase) ?? false)
            || (p.IdCategoryNavigation?.Name?.Contains(_searchText, StringComparison.OrdinalIgnoreCase) ?? false);
        bool f = _filterSupplierId == null || p.IdSupplier == _filterSupplierId;
        return s && f;
    };
}
```
  Сортировка — `view.SortDescriptions.Add(new SortDescription("Quantity", ...))`; совмещается с фильтром на том же view.
- **Форма (одно окно на добавление и редактирование):** конструктор `ProductEditWindow(context, product = null)`; `product == null` → добавление; `DataContext = product`; на Save — валидация + (`if isNew` `Add`) + `SaveChanges` + `DialogResult = true`. Открытие: `if (win.ShowDialog() == true) ReloadProducts();`
- **Удаление с проверкой:** `_context.OrderItems.Any(oi => oi.IdProduct == p.Id)` → если да, MessageBox и выход.
- **Вход:** искать по логину И паролю (`Include(u => u.IdRoleNavigation)`), `MessageBox` при неудаче, `Navigate` только при успехе; гость = `CurrentUser = null`.

## 7. Разобранные грабли (важно не повторять)

- **`ListBox`/грид НЕ оборачивать в `ScrollViewer`** — у них свой скролл; внешний даёт бесконечную высоту и контент вылезает. Список — в строку `*`, поиск — в строку `Auto`.
- **`Width="*"` у элемента не работает** — звёздочка только у `ColumnDefinition`/`RowDefinition`. Для «во всю ширину» — без `Width` + `HorizontalAlignment="Stretch"`; карточки во всю ширину — `ListBox HorizontalContentAlignment="Stretch"`.
- **Подмена контента:** `MainContent.Content = view`, а НЕ `MainContent = view`.
- **`UserControl` vs `Page`:** Page в ContentControl даёт `ParentOfPageMustBeWindowOrFrame`. Все экраны — `UserControl`. У `UserControl` нет `Title`.
- **`Window.GetWindow(this)` в конструкторе = null** (контрол ещё не в дереве) → ловить окно через `Session.Window`, а не из конструктора.
- **Конструктор ничего не возвращает** — для настройки контекста использовать `: base(options)` или `OnConfiguring`, а не `return ctx;`.
- **`PasswordBox` читается через `.Password`**, не `.Text`, и не биндится.
- **ObservableCollection ≠ List** — для привязки и живого обновления нужен `ObservableCollection` (а `Where(...)` возвращает `IEnumerable`, обратно не присвоить).
- **Редактирование+сохранение работают без `INotifyPropertyChanged`** (биндинг пишет в объект, EF сам трекает). INPC нужен только для живого обновления UI из кода.
- **`[ObservableProperty]`** вешается на **поле** (не на свойство); DataAnnotations пробрасывать через `[property: ...]`. Для заголовков/подписей использовать `[Display(Name=...)]` (а не `[DisplayName]`).
- **Подсветка по условию** — через `IValueConverter`, привязка фона к самому объекту: `Background="{Binding Converter={StaticResource ...}}"`.
- **Клик по «пустому» месту карточки не ловится**, если у контейнера нет фона → `Background="Transparent"`; объект клика брать из `(sender as FrameworkElement).DataContext`.
